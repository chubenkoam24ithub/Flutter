## Описание проекта

**CineMate** – это мобильное приложение-социальная сеть для киноманов. Проект позволяет пользователям искать информацию о фильмах, вести свой профиль, делиться впечатлениями от просмотра и читать отзывы других пользователей в общей ленте.

Проект демонстрирует уверенные навыки разработки кроссплатформенных мобильных приложений на Flutter, построение чистой архитектуры, интеграцию со сторонними REST API и использование облачных сервисов.

## Архитектурные особенности и стек технологий

- **Фреймворк:** Flutter (Dart).  
- **Архитектура и управление состоянием:** BLoC (Business Logic Component). Приложение разбито на независимые слои: UI (экраны), BLoC (бизнес-логика), Repositories (работа с данными), Models (модели данных).  
- **Сетевое взаимодействие:** Пакет `http`. Реализовано обращение к открытому REST API базы данных кино (OMDb API) с последующей десериализацией JSON в модели Dart.  
- **Облачные технологии:** Firebase (Firebase Storage). Используется для загрузки, хранения и получения фотографий профиля пользователя (аватарок).  
- **Локальное хранилище:** `shared_preferences`. Применяется для сохранения пользовательских настроек (выбор светлой/темной темы, параметров уведомлений).  
- **UI/UX и Навигация:**
  - Использование Material Design (Scaffold, BottomNavigationBar, Card, CircleAvatar, AlertDialog, TextField, SwitchListTile и др.).
  - Глобальная смена темы (Dark/Light Mode) без перезагрузки приложения с использованием `ThemeData`.
  - Маршрутизация с передачей аргументов (Named Routes) между экранами профилей.

## Функциональность

- **Поиск фильмов:** Интеллектуальный поиск фильмов по названию через OMDb API с выводом постера, года выпуска, рейтинга IMDb и краткого сюжета.  
- **Профиль пользователя:** Возможность выбрать фотографию из галереи устройства (`image_picker`), загрузить её в Firebase и опубликовать отзыв на найденный фильм.  
- **Глобальная лента:** Просмотр ленты рецензий от других (mock) пользователей с отображением оценок фильмов.  
- **Детальный просмотр:** Переход на страницу конкретного пользователя из ленты для изучения его кино-предпочтений.  
- **Настройки:** Управление темой приложения и системными переключателями, а также логика выхода из профиля.

## Задание

В рамках дисциплины «Flutter» требовалось разработать кроссплатформенное мобильное приложение, удовлетворяющее следующим критериям:

1. Клиентская часть на Flutter (Dart) с использованием одного из популярных подходов к управлению состоянием (BLoC).  
2. Интеграция со сторонним REST API (OMDb API) для получения данных о фильмах.  
3. Использование облачного сервиса (Firebase Storage) для хранения пользовательских изображений.  
4. Локальное сохранение настроек приложения через `shared_preferences`.  
5. Реализация основных экранов: поиск фильмов, профиль пользователя, глобальная лента, настройки.  
6. Система контроля версий (Git) и хостинг исходного кода на GitHub.
